Создай новый документ в проектной документации Benovite.

## Инструкции

1. Прочитай docs/README.md для понимания текущей структуры
2. Определи правильную папку для нового документа:
   - 00_strategy/ — стратегия, планирование, чеклисты
   - 01_brand/ — бренд, визуал, tone of voice
   - 02_data/ — источники данных, пайплайны, схемы БД
   - 03_product/ — продукт, шаблоны, UX, i18n
   - 04_seo/ — SEO, schema.org, ключевые слова
   - 05_monetization/ — модели дохода, pricing
   - 06_tech/ — инфраструктура, деплой, безопасность
   - 07_legal/ — GDPR, disclaimers, комплаенс
3. Создай файл с правильной структурой:
   - Заголовок
   - Статус и дата обновления
   - Содержание
   - TODO секция в конце
4. Обнови docs/README.md — добавь ссылку на новый документ
5. Обнови CLAUDE.md если документ влияет на структуру проекта

## Формат

```markdown
# Название документа

> Статус: черновик / шаблон / финализировано
> Последнее обновление: YYYY-MM-DD

---

## Содержание
...

---

## TODO
- [ ] ...
```

## Аргумент
$ARGUMENTS — тема или название документа (например: "api design", "database schema", "deployment guide")
